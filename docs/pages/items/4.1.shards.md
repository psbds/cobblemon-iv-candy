## Drop Mechanics

### Drop Priority System

When a Pok√©mon faints, the mod checks for drops in this order:
When a Pok√©mon faints, the mod checks for drops in this order:

1. **Special category drops** (Legendary/Mythical/Ultra Beast/Paradox) - these override species drops
2. **Type-specific drops** - checked for all Pok√©mon regardless of category
3. **Species drops** - only for regular (non-special) Pok√©mon

### Dual-Type Pok√©mon Logic

Pok√©mon with dual types have separate chances to drop shards for each of their types:
- **Independent Rolls**: Each type is rolled separately
- **Multiple Drops**: A single Pok√©mon can potentially drop multiple type shards
- **Example**: A Fire/Flying Pok√©mon can drop both Fire and Flying shards in one encounter

### Drop Requirements

- **Pok√©mon Must Faint**: Only fully fainted Pok√©mon trigger shard drops
- **Any Source**: Works with wild Pok√©mon, owned Pok√©mon, and battle encounters
- **Automatic Collection**: Shards appear as item drops that can be picked up
- **No Player Requirement**: Drops occur regardless of who owns the Pok√©mon
